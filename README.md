# tp1
### ¿Por qué es conveniente incluirlo?
Incluir un .gitignore en tu proyecto es muy conveniente porque evita subir a Git archivos que no deberían formar parte del repositorio. Eso te ahorra problemas y mantiene todo más limpio.
### ¿Cuándo se debe hacer?
Lo ideal es crear el .gitignore lo antes posible, preferentemente antes del primer commit o al agregar una nueva tecnologia
### ¿Cómo configuraría el archivo .gitignore?
Para configurar el archivo .gitignore, debes crearlo en la raíz de tu repositorio (o en el directorio deseado) con el nombre exacto .gitignore y añadir patrones de exclusión en cada línea.
